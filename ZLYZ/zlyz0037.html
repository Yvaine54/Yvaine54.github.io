Git bash 命令

    编号：0037
    日期：2020.05.08    分类：杂乱有章

本来说37这篇写碎碎念的哈哈...不好意思，我暂时还没时间

这是去年七月记的东西了...一直放在桌面没时间发。

【ls】查看当前文件

【pwd】查看当前文件目录

【git init】初始化，会生成一个.git目录

【clear】清屏

【git config --global user.name “github用户名”】设置用户名信息(注意使用双引号，以及前方空格)

【git config --global user.email “邮箱地址”】设置用户名邮箱(注意使用双引号，以及前方空格)

【mkdir 文件夹名】创建文件夹

【cd 文件夹名】进入该文件夹

【git init】在文件夹内初始化git（创建git仓库）

【touch 文件名.文件格式】创建文件

【git status】查看文件状态

【git add 文件名.文件格式】提交到暂存区域

【git commit -m ‘add 文件名.文件格式’】继续提交到仓库

【git config --list】查看git表单

【git config user.email】查看git邮箱设置

【git config user.email】查看git用户名设置


【Q键】退出list


修改仓库文件

【vi 文件名.文件格式】打开要编辑文件

【键盘“Esc”键 再“Insert”键】调整为插入模式 即可修改

【键盘“Esc”键 输入“：q！”】不保存文件，强制退出vi命令

【键盘“Esc”键 输入“：w”】保存文件，不退出vi命令

【键盘“Esc”键 输入“：wq”】保存文件，退出vi命令

【git add 文件名.文件格式】提交到暂存区域

【git commit -m ‘添加自定义描述’】继续提交到仓库


删除仓库文件

【rm -rf 文件名.文件格式】删除文件

【git rm 文件名.文件格式】从暂存区删除文件

【git commit -m ‘描述’】提交修改

提交到github仓库

【git clone 仓库地址】克隆仓库到本地

【git add 文件名.文件格式】提交到暂存区域

【git commit -m ‘添加自定义描述’】继续提交到仓库

【git push】提交到云仓库


修改文件

先在本地修改名字

【git rm原文件名.文件格式】删除暂存盘内的原文件

【git add 新文件名.文件格式】添加新文件到暂存盘

【git commit -m ‘添加描述’】提交到本地仓库

【git push】提交到云仓库
